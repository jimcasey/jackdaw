**Slice 4 spec = `docs/slices/slice-4-triage.md`** (tech-lead, 2026-07-14). The "real Triage inbox." `TriageRootView` GROWS from the read-only @Query list (it is NOT rewritten). Simulator-sufficient. Prereq: sheet-model capture rework done (18 tests green); `CaptureService`/`CaptureViewModel`/`Note`/`TriageRootView` exist in `Jackdaw/` (flat, not in subfolders except `Talon/`).

**NUMBERING drift:** implementation numbering runs ONE AHEAD of `docs/build-order.md`'s at-a-glance table (the sheet-capture rework became its own slice). This is THE triage slice regardless of index; precedes Location → Apple Notes → Obsidian.

**1. Note model extension (additive → lightweight migration; ADR 0003 plan):** add to `Note`: `private var statusRaw: String = NoteStatus.inbox.rawValue`, `var snoozedUntil: Date?`, `var snoozeCount: Int = 0`, computed `var status: NoteStatus`. **Enum:** `enum NoteStatus: String, Codable, CaseIterable { case inbox, snoozed, kept }`. Export slices EXTEND with pending/writing/confirmed (adding an enum case is NOT a schema change — stored col is String — no migration; don't fork a 2nd enum). **Store raw String + computed enum** because `#Predicate` reliably compares stored primitives but is historically flaky with custom enum types (raw-string is the robust SwiftData pattern; also debuggable in-store). `.kept` notes accumulate held (no export pipeline yet) — correct, not lost.

**Reconciliation (2 orthogonal axes — do NOT conflate):** `status` = WHERE (inbox→snoozed→kept→[pending→writing→confirmed→deleted at export]); `exportFailure: ExportFailure?` (added at EXPORT slice, not now) = WHY stuck, reusing the Talon enum `Jackdaw/Talon/ExportDestination.swift` (noVaultConfigured/accessLost/writeFailed/verifyMismatch, +`.offline` at export). "kept-but-no-destination" = status `.pending` + `exportFailure .noVaultConfigured`. Separating state from reason is what lets Settings/Status UI pick Retry vs Re-grant vs Set-up-vault.

**2. Actions→transitions:** Keep = `status=.kept` (leading full-swipe). Snooze = `status=.snoozed` + set snoozedUntil + `snoozeCount+=1` (trailing amber). Discard = delete, hard or deferred per undo decision (trailing full-swipe destructive). Tap row→editor; long-press→context menu (accessible mirror; swipe actions also need VoiceOver custom actions).

**3. SNOOZE MODEL (concrete):** the CALENDAR-DAY boundary IS the session boundary → NO session-id field needed. Store `snoozedUntil` = start of NEXT local calendar day, computed once at snooze time (so the query needs no Calendar, which can't run in #Predicate). `SnoozeSchedule.reappearBoundary(snoozedAt:calendar:) = startOfDay(snoozedAt) + 1 day`; `isDue(snoozedUntil:now:) = now >= snoozedUntil`. Calendar+now INJECTABLE for tests. snoozeCount surfaces "snoozed N×" at N>=3 (nudge, not hard-disable). Midnight edge (snooze 11:58pm → back ~2min) accepted; drop-in refinement "next 4am or ≥6h whichever later" lives in the one pure function, not pre-built.

**4. DISCARD-UNDO fork — RECOMMEND ADOPT (framed owner's call). Position CHANGED:** previously resisted (cost = persistent tombstone/purge); with the **deferred-delete-in-memory** impl that cost is GONE (no schema change, transient VM state only), and design's autosave-consistency argument holds (every note is real data → mis-tap Discard destroys a genuine thought; autosave made loss impossible at capture, don't reintroduce it at triage). **Impl (if adopted):** VM holds `pendingDiscard: Set<UUID>` + `held: [UUID:Note]`; `discard` HIDES only (optimistic, no delete); `undoDiscard` restores; `commitDiscard` (banner expiry) does `context.delete`+save. `visibleNotes` filters pendingDiscard. **Prefer deferred-delete over `.discarded` soft-delete tombstone:** no purge policy, and KILL-SAFE IN FUNNEL'S FAVOR (app killed mid-window → delete never fires → note survives; a tombstone would make discard durable across kill = wrong bias). **Semantics follow the decision:** undo adopted → Discard = deferred hard-delete (~3-5s window); undo rejected → immediate hard-delete. Banner = custom transient view (iOS has no snackbar) — system materials, safe areas, VoiceOver announce + accessible Undo (design owns).

**5. Light editing:** tap row → PUSH editor (design chose push). Ship BODY editing (full-bleed TextEditor bound to note.body, autosave + save-on-disappear, PLAIN TEXT only — no markdown, PRD non-goal). Three verbs in bottom bar → same VM actions. TIME editing = recommend include (DatePicker on existing createdAt, in-scope, cheap; owner may defer). LOCATION editing = deferred to Location slice (fields don't exist); leave a Context-section hook. Editor does NOT prune emptied body (prune is capture-only).

**6. TriageRootView grows (don't rewrite):** wrap in `NavigationStack`. `@Query` fetches CANDIDATES (predicate `statusRaw==inbox || statusRaw==snoozed`, primitive). Due-filter is a PURE VM function `visibleNotes` (NOT in the predicate — keeps calendar/now injectable/testable, not frozen at view-init); recompute on appear/scenePhase active (no live timer; due-ness changes only at day rollover). List+swipe on visibleNotes; title count = visibleNotes.count (excludes not-yet-due snoozed); grow ContentUnavailableView "Inbox clear" + count-only "N will return in a later session" (no browse affordance). MVVM: reads @Query, writes/due-filter in `@Observable TriageViewModel` (injected ModelContext + injected clock).

**7. Testing:** off-device in-memory container — transitions; snooze due-logic with INJECTED clock+calendar (day rollover 3pm→11pm not due→9am next day due; boundary exact; timezone); inbox filter (inbox+due-snoozed only); undo (discard hides-not-deletes, undo restores, commit deletes) testing commit/undo as direct calls NOT the wall-clock timer. NEVER call Date() in assertion path. Simulator: keep/discard+undo/snooze(advance device date to verify reappear)/edit/empty-state.
